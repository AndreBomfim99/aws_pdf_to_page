Descrição das Responsabilidades Compartilhadas entre Cliente e AWS

O Modelo de Responsabilidade Compartilhada da AWS define as responsabilidades conjuntas entre a AWS e seus clientes. Enquanto a AWS gerencia a infraestrutura física e os serviços gerenciados, os clientes são responsáveis por configurações seguras e o gerenciamento de suas aplicações e dados.

O Conceito de Responsabilidade Compartilhada

As responsabilidades compartilhadas envolvem a interseção entre a segurança da nuvem (AWS) e a segurança na nuvem (cliente). Isso garante a segurança geral dos serviços utilizados.

Principais Responsabilidades Compartilhadas

Responsabilidade Responsabilidade da AWS Responsabilidade do Cliente
Segurança Física A AWS protege os data centers e toda a infraestrutura física. Sem responsabilidade direta do cliente.
Gerenciamento de Sistema Operacional Se for um serviço gerenciado (ex.: RDS), a AWS gerencia o sistema subjacente. Se usar EC2, o cliente gerencia o sistema operacional hospedado na instância.
Atualização de Patches A AWS aplica patches no hardware e nos serviços gerenciados. O cliente aplica patches em sistemas operacionais e aplicações instaladas.
Segurança de Rede A AWS protege a infraestrutura de rede subjacente. O cliente configura grupos de segurança, VPCs e ACLs para proteger o tráfego de rede.
Gerenciamento de Dados A AWS fornece serviços de armazenamento seguros. O cliente protege, gerencia e faz backup de seus dados armazenados.

Exemplo Prático

Suponha que você utilize o Amazon EC2 para hospedar uma aplicação:

  • AWS: Gerencia a infraestrutura física, como data centers e redes.
  • Cliente: Configura o sistema operacional, aplica patches, implementa grupos de segurança e protege os dados da aplicação.

Benefícios do Modelo Compartilhado

  • Flexibilidade: Clientes podem implementar controles adicionais conforme suas necessidades.
  • Redução de Tarefas Operacionais: A AWS gerencia partes complexas da infraestrutura.
  • Segurança Colaborativa: A cooperação entre cliente e AWS garante um ambiente mais seguro.

Conclusão

No modelo de responsabilidade compartilhada, tanto a AWS quanto os clientes têm papéis importantes na segurança e na gestão de infraestrutura e serviços. Compreender essas responsabilidades é essencial para operar com segurança e eficiência na nuvem.
