Mudança das Responsabilidades da AWS e do Cliente de Acordo com o Serviço Usado

As responsabilidades da AWS e do cliente variam conforme o serviço utilizado. A AWS fornece diferentes níveis de gerenciamento, desde infraestruturas totalmente gerenciadas até serviços que exigem maior controle do cliente.

Variação das Responsabilidades

O Modelo de Responsabilidade Compartilhada da AWS adapta-se aos diferentes tipos de serviços:

  • Infraestrutura como Serviço (IaaS): Mais responsabilidades para o cliente (ex.: Amazon EC2).
  • Plataforma como Serviço (PaaS): A AWS gerencia mais componentes (ex.: Amazon RDS).
  • Função como Serviço (FaaS): A AWS gerencia quase tudo, enquanto o cliente foca apenas no código (ex.: AWS Lambda).

Exemplos de Mudança de Responsabilidades

Serviço Responsabilidade da AWS Responsabilidade do Cliente
Amazon EC2 (IaaS) Gerenciar infraestrutura física, rede e hipervisor. Configurar sistema operacional, aplicar patches, gerenciar aplicações e dados.
Amazon RDS (PaaS) Gerenciar infraestrutura, sistema operacional, instalação e patches do banco de dados. Configurar parâmetros do banco de dados, gerenciar acesso e proteger os dados.
AWS Lambda (FaaS) Gerenciar infraestrutura, escalabilidade, segurança do ambiente de execução. Fornecer código da função, lógica da aplicação e configurar triggers.
Amazon S3 Gerenciar infraestrutura, alta disponibilidade e durabilidade dos dados. Configurar políticas de acesso, criptografar dados e monitorar atividades.

Resumo da Distribuição

A figura a seguir ilustra como a responsabilidade se desloca entre a AWS e o cliente com base no nível de serviço:

  • IaaS: Mais controle e responsabilidade do cliente.
  • PaaS: Gerenciamento compartilhado, com menos responsabilidades para o cliente.
  • FaaS: A AWS gerencia quase tudo; o cliente se preocupa apenas com o código.

Exemplo Prático

Suponha que você desenvolva uma aplicação usando:

  • Amazon EC2: Você instala o sistema operacional, aplica patches e gerencia o servidor.
  • Amazon RDS: A AWS cuida da infraestrutura e das atualizações; você configura e gerencia o banco de dados.
  • AWS Lambda: Você carrega apenas o código da aplicação, e a AWS gerencia toda a infraestrutura.

Conclusão

As responsabilidades entre AWS e cliente mudam conforme o serviço utilizado. Compreender essas variações é fundamental para usar os serviços da AWS de maneira eficiente, segura e alinhada com as necessidades do negócio.
